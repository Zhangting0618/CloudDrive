# 事务管理使用指南

## 概述

本项目提供了一套完整的事务管理方案，支持以下功能：

1. **自动事务管理** - 使用 `[Transactional]` 特性自动处理事务提交和回滚
2. **独立提交** - 使用 `[IndependentSubmit]` 标记某些操作不受事务回滚影响
3. **手动事务控制** - 使用 `IUnitOfWork` 手动控制事务边界

## 使用方式

### 方式一：声明式事务管理（推荐）

在需要事务的 Command 类上添加 `[Transactional]` 特性：

```csharp
[Transactional]
public class CreateFolderCommandHandler : IRequestHandler<CreateFolderRequestDto, ResponseMessageDto<bool>>
{
    private readonly IFileRepository fileRepository;
    private readonly IUserRepository userRepository;

    public CreateFolderCommandHandler(IFileRepository fileRepository, IUserRepository userRepository)
    {
        this.fileRepository = fileRepository;
        this.userRepository = userRepository;
    }

    public async Task<ResponseMessageDto<bool>> Handle(CreateFolderRequestDto request, CancellationToken cancellationToken)
    {
        // 如果这个方法中任何地方抛出异常，整个事务会自动回滚
        // 如果完全没错误，事务会自动提交

        var fileEntity = new FileEntity();
        // ... 设置属性

        // 这里的数据库操作会在事务中执行
        await fileRepository.SaveFileEntity(fileEntity, await userRepository.UserId(), true);

        return new ResponseMessageDto<bool> { IsSuccess = true, Message = "创建成功" };
    }
}
```

**特性参数说明：**

```csharp
[Transactional(
    IsolationLevel = IsolationLevel.ReadCommitted,  // 隔离级别，默认 ReadCommitted
    TimeoutSeconds = 30,                             // 超时时间（秒），默认 30 秒
    Enabled = true                                   // 是否启用事务，默认 true
)]
```

### 方式二：独立提交（不受回滚影响）

如果你希望某些数据库操作**不受外层事务影响**，即使外层事务回滚，这些操作也会提交：

#### 场景 1：日志记录

```csharp
[Transactional]
public class ProcessFileCommandHandler : IRequestHandler<ProcessFileRequestDto, ResponseMessageDto<bool>>
{
    private readonly IFileRepository fileRepository;
    private readonly IRepository<OperationLogEntity> logRepository;

    public async Task<ResponseMessageDto<bool>> Handle(ProcessFileRequestDto request, CancellationToken cancellationToken)
    {
        try
        {
            // 文件处理逻辑 - 失败会回滚
            await fileRepository.AddAsync(fileEntity, cancellationToken);

            // 记录操作日志 - 即使外层事务回滚，日志也会提交
            await logRepository.AddIndependentAsync(new OperationLogEntity
            {
                Action = "处理文件",
                Timestamp = DateTime.Now,
                Success = true
            }, cancellationToken);

            return new ResponseMessageDto<bool> { IsSuccess = true };
        }
        catch (Exception ex)
        {
            // 文件操作会回滚，但上面的日志已经提交了
            // 这里可以记录错误日志，同样会独立提交
            await logRepository.AddIndependentAsync(new OperationLogEntity
            {
                Action = "处理文件失败",
                Error = ex.Message,
                Timestamp = DateTime.Now,
                Success = false
            }, cancellationToken);

            throw; // 抛出异常会触发事务回滚
        }
    }
}
```

#### 场景 2：进度记录

```csharp
[Transactional]
public class BatchImportCommandHandler : IRequestHandler<BatchImportRequestDto, ResponseMessageDto<bool>>
{
    private readonly IRepository<ImportProgressEntity> progressRepository;
    private readonly IFileRepository fileRepository;

    public async Task<ResponseMessageDto<bool>> Handle(BatchImportRequestDto request, CancellationToken cancellationToken)
    {
        int total = request.Files.Count;
        int processed = 0;

        foreach (var file in request.Files)
        {
            // 导入文件 - 失败会回滚
            await fileRepository.AddAsync(file.ToEntity(), cancellationToken);
            processed++;

            // 更新进度 - 独立提交，即使后面失败，已处理的进度也会保留
            await progressRepository.UpdateIndependentAsync(new ImportProgressEntity
            {
                BatchId = request.BatchId,
                Processed = processed,
                Total = total,
                LastUpdateTime = DateTime.Now
            }, cancellationToken);
        }

        return new ResponseMessageDto<bool> { IsSuccess = true };
    }
}
```

### 方式三：手动控制事务

使用 `IUnitOfWork` 手动控制事务边界：

```csharp
public class ComplexOperationCommandHandler : IRequestHandler<ComplexOperationRequestDto, ResponseMessageDto<bool>>
{
    private readonly IUnitOfWork unitOfWork;
    private readonly IFileRepository fileRepository;

    public ComplexOperationCommandHandler(IUnitOfWork unitOfWork, IFileRepository fileRepository)
    {
        this.unitOfWork = unitOfWork;
        this.fileRepository = fileRepository;
    }

    public async Task<ResponseMessageDto<bool>> Handle(ComplexOperationRequestDto request, CancellationToken cancellationToken)
    {
        // 手动开始事务
        using var transaction = await unitOfWork.BeginTransactionAsync(
            isolationLevel: System.Data.IsolationLevel.ReadCommitted,
            cancellationToken: cancellationToken);

        try
        {
            // 业务逻辑
            await fileRepository.AddAsync(fileEntity1, cancellationToken);
            await fileRepository.AddAsync(fileEntity2, cancellationToken);

            // 提交事务
            await unitOfWork.CommitAsync(cancellationToken);

            return new ResponseMessageDto<bool> { IsSuccess = true };
        }
        catch (Exception ex)
        {
            // 回滚事务
            await unitOfWork.RollbackAsync(cancellationToken);
            throw;
        }
    }
}
```

### 方式四：标记实体为独立提交

如果你希望某个实体类型的所有操作都独立提交：

```csharp
// 标记操作日志实体为独立提交
[IndependentSubmitEntity(Description = "操作日志，始终提交不受事务影响")]
public class OperationLogEntity
{
    public long Id { get; set; }
    public string Action { get; set; }
    public DateTime Timestamp { get; set; }
    // ...
}
```

这样，对 `OperationLogEntity` 的所有操作都会自动独立提交。

## 事务隔离级别说明

| 隔离级别 | 脏读 | 不可重复读 | 幻读 | 说明 |
|---------|------|-----------|------|------|
| ReadUncommitted | 可能 | 可能 | 可能 | 允许读取未提交的数据，性能最高 |
| ReadCommitted | 防止 | 可能 | 可能 | 只能读取已提交的数据（**默认推荐**） |
| RepeatableRead | 防止 | 防止 | 可能 | 防止脏读和不可重复读 |
| Serializable | 防止 | 防止 | 防止 | 最高隔离级别，性能最低 |
| Snapshot | 防止 | 防止 | 防止 | 使用版本控制，适合高并发读 |

## 最佳实践

### 1. 事务粒度

```csharp
// 推荐：小事务，快速提交
[Transactional]
public class CreateUserCommandHandler : IRequestHandler<...>
{
    public async Task<...> Handle(...)
    {
        // 只处理用户创建，快速提交
        await userRepository.AddAsync(user);
    }
}

// 不推荐：大事务，包含远程调用
[Transactional]
public class ComplexProcessCommandHandler : IRequestHandler<...>
{
    public async Task<...> Handle(...)
    {
        await userRepository.AddAsync(user);
        await fileRepository.AddAsync(file);

        // 不推荐：在事务中调用外部 API
        await externalService.SendData(); // 可能超时导致事务长时间不提交

        await logRepository.AddAsync(log);
    }
}
```

### 2. 异常处理

```csharp
// 推荐：让异常自然传播，触发回滚
[Transactional]
public class CreateFileCommandHandler : IRequestHandler<...>
{
    public async Task<...> Handle(...)
    {
        // 验证失败直接抛出异常，事务自动回滚
        if (string.IsNullOrEmpty(fileName))
            throw new ArgumentException("文件名不能为空");

        await fileRepository.AddAsync(file);
        return success;
    }
}

// 不推荐：捕获异常后吞掉，导致事务提交
[Transactional]
public class BadExampleCommandHandler : IRequestHandler<...>
{
    public async Task<...> Handle(...)
    {
        try
        {
            await fileRepository.AddAsync(file);
        }
        catch (Exception ex)
        {
            // 错误：捕获异常后返回错误信息，事务会提交！
            return new ResponseMessageDto<bool> { IsSuccess = false, Message = ex.Message };
        }
        return success;
    }
}
```

### 3. 独立提交的使用场景

**适合使用独立提交的场景：**
- 操作日志记录
- 审计跟踪
- 进度记录
- 监控数据
- 临时状态保存

**不适合使用独立提交的场景：**
- 核心业务数据
- 需要保证原子性的操作
- 与其他数据有关联的数据

## 完整示例

```csharp
using Ptcent.Cloud.Drive.Application.Attributes;
using Ptcent.Cloud.Drive.Application.Interfaces;
using Ptcent.Cloud.Drive.Application.Interfaces.Persistence;

namespace Ptcent.Cloud.Drive.Application.Handlers.CommandHandlers.File
{
    /// <summary>
    /// 文件上传命令处理器 - 带事务管理
    /// </summary>
    [Transactional(IsolationLevel = IsolationLevel.ReadCommitted, TimeoutSeconds = 60)]
    public class UploadFileCommandHandler : IRequestHandler<UploadFileRequestDto, ResponseMessageDto<bool>>
    {
        private readonly IFileRepository fileRepository;
        private readonly IRepository<FileUploadLogEntity> logRepository;
        private readonly IUnitOfWork unitOfWork;

        public UploadFileCommandHandler(
            IFileRepository fileRepository,
            IRepository<FileUploadLogEntity> logRepository,
            IUnitOfWork unitOfWork)
        {
            this.fileRepository = fileRepository;
            this.logRepository = logRepository;
            this.unitOfWork = unitOfWork;
        }

        public async Task<ResponseMessageDto<bool>> Handle(UploadFileRequestDto request, CancellationToken cancellationToken)
        {
            try
            {
                // 1. 创建文件记录（在事务中）
                var fileEntity = new FileEntity
                {
                    // ... 设置属性
                };
                await fileRepository.AddAsync(fileEntity, cancellationToken);

                // 2. 记录上传日志（独立提交，即使后面失败日志也会保留）
                await logRepository.AddIndependentAsync(new FileUploadLogEntity
                {
                    FileId = fileEntity.Id,
                    UploadTime = DateTime.Now,
                    Status = "Processing",
                    Message = "文件上传处理中"
                }, cancellationToken);

                // 3. 模拟一些可能失败的操作
                if (request.FileSize > 1024 * 1024 * 100) // 大于 100MB
                {
                    throw new InvalidOperationException("文件大小超过限制");
                }

                // 4. 更新文件状态（在事务中）
                fileEntity.Status = 1;
                await fileRepository.UpdateAsync(fileEntity, cancellationToken);

                return new ResponseMessageDto<bool>
                {
                    IsSuccess = true,
                    Message = "文件上传成功"
                };
            }
            catch (Exception ex)
            {
                // 异常会被 TransactionBehavior 捕获，触发事务回滚
                // 但上面的日志已经独立提交了

                // 可以在这里记录错误日志（同样会独立提交）
                await logRepository.AddIndependentAsync(new FileUploadLogEntity
                {
                    UploadTime = DateTime.Now,
                    Status = "Failed",
                    Message = $"处理失败：{ex.Message}"
                }, cancellationToken);

                return new ResponseMessageDto<bool>
                {
                    IsSuccess = false,
                    Message = $"文件上传失败：{ex.Message}"
                };
            }
        }
    }
}
```

## 注意事项

1. **事务超时**：默认 30 秒超时，超时后会自动回滚
2. **嵌套事务**：如果已有活动事务，新的 `[Transactional]` 会加入现有事务
3. **独立提交原理**：独立提交通过创建新的事务来实现，所以即使外层回滚，独立提交的数据也会保留
4. **性能考虑**：频繁使用独立提交可能影响性能，建议只在必要时使用
