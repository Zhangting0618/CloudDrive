## 使用方式

### 方式 1：自动事务（推荐，默认）

在 `Program.cs` 中注册：

```csharp
services.AddApplicationServices(
    TransactionMode.Automatic); // 默认值，可省略
```

所有 Command 都会自动添加事务支持，无需额外标记。

### 方式 2：显式事务

只有标记了 `[Transactional]` 特性的 Command 才有事务：

```csharp
services.AddApplicationServices(
    TransactionMode.Explicit);
```

```csharp
[Transactional(IsolationLevel = IsolationLevel.ReadCommitted)]
public class CreateFileCommandHandler : IRequestHandler<...>
{
    // ...
}
```

## 三种场景的处理

### 场景 1：接口抛出异常，回滚事务

```csharp
[Transactional] // 或使用 Automatic 模式
public class CreateFileCommandHandler : IRequestHandler<...>
{
    public async Task<ResponseMessageDto<bool>> Handle(...)
    {
        // 如果这里抛出异常，事务会自动回滚
        await fileRepository.AddAsync(fileEntity);
        throw new Exception("失败了"); // 事务回滚
    }
}
```

### 场景 2：接口完全没错误，事务提交

```csharp
[Transactional]
public class CreateFileCommandHandler : IRequestHandler<...>
{
    public async Task<ResponseMessageDto<bool>> Handle(...)
    {
        // 没有异常，事务自动提交
        await fileRepository.AddAsync(fileEntity);
        return new ResponseMessageDto<bool> { IsSuccess = true };
    }
}
```

### 场景 3：某些表的提交不受影响（独立提交）

```csharp
[Transactional]
public class ProcessFileCommandHandler : IRequestHandler<...>
{
    private readonly IFileRepository fileRepository;
    private readonly IRepository<OperationLogEntity> logRepository;

    public async Task<ResponseMessageDto<bool>> Handle(...)
    {
        try
        {
            // 文件操作 - 失败会回滚
            await fileRepository.AddAsync(fileEntity);

            // 日志记录 - 即使外层失败，这行也会提交
            await logRepository.AddIndependentAsync(new OperationLogEntity { ... });

            throw new Exception("模拟失败"); // 文件操作回滚，但日志已提交

            return new ResponseMessageDto<bool> { IsSuccess = true };
        }
        catch
        {
            // 记录错误日志 - 同样会独立提交
            await logRepository.AddIndependentAsync(new OperationLogEntity
            {
                Action = "处理失败",
                Error = ex.Message
            });
            throw;
        }
    }
}
```

## 独立提交的其他方式

### 方式 1：使用 Repository 的独立提交方法

```csharp
await repository.AddIndependentAsync(entity);
await repository.UpdateIndependentAsync(entity);
await repository.DeleteIndependentAsync(entity);
```

### 方式 2：标记实体为独立提交

```csharp
[IndependentSubmitEntity(Description = "操作日志，始终提交")]
public class OperationLogEntity
{
    // ...
}
```

### 方式 3：使用 IUnitOfWork 手动控制

```csharp
public class MyHandler : IRequestHandler<...>
{
    private readonly IUnitOfWork unitOfWork;

    public async Task Handle(...)
    {
        using var transaction = await unitOfWork.BeginTransactionAsync();
        try
        {
            // 业务逻辑
            await unitOfWork.CommitAsync();
        }
        catch
        {
            await unitOfWork.RollbackAsync();
            throw;
        }
    }
}
```

## 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                      MediatR Pipeline                        │
├─────────────────────────────────────────────────────────────┤
│  1. ValidationBehavior (验证)                                │
│  2. TransactionBehavior / AutomaticTransactionBehavior (事务)│
│  3. Handler (业务逻辑)                                       │
└─────────────────────────────────────────────────────────────┘

事务处理流程：

正常流程：
Request → BeginTransaction → Handler → SaveChanges → Commit → Response

异常流程：
Request → BeginTransaction → Handler throws → Rollback → Exception

独立提交流程：
Request → BeginTransaction → Handler → IndependentSave (立即提交)
          ↓
      后续失败回滚 ← 独立提交的数据不受影响
```

## 注意事项

1. **事务超时**：默认 30 秒，超时后自动回滚
2. **嵌套事务**：如果已有活动事务，新的请求会加入现有事务
3. **独立提交原理**：通过创建新事务实现，所以不受外层回滚影响
4. **性能**：独立提交频繁使用可能影响性能
